# Contributing to the SALE

Thank you for your interest in improving the SALE. This project is an open-source legal instrument, not a software project, so contributions look a little different than a typical GitHub repo. Here's how you can help.

## Ways to Contribute

### 1. Report an Issue

If you spot a drafting error, ambiguity, missing provision, or jurisdictional concern, [open an Issue](https://github.com/brazensimulacrum/SALE/issues/new/choose). Please include:

- **Which section** of the SALE is affected (e.g., "Section 4(b)(iii)")
- **What the problem is** (e.g., "Undefined term," "Inconsistent cross-reference," "Missing standard rep")
- **Suggested fix**, if you have one

### 2. Suggest a Substantive Change

For proposed changes to the operative terms of the SALE — new provisions, revised definitions, alternative structures — please open an Issue first to discuss the rationale before submitting a Pull Request. Legal instruments require careful consideration of how changes interact with other provisions, so discussion before drafting is encouraged.

### 3. Propose Supporting Documentation

The SALE ecosystem benefits from companion materials: user guides, worked examples, jurisdiction-specific annotations, board resolution templates, and similar resources. If you'd like to contribute something along these lines, open an Issue describing what you have in mind.

### 4. Join the Discussion

The [Discussions](https://github.com/brazensimulacrum/SALE/discussions) tab is open for general conversation about the SALE — use cases, adoption experiences, comparisons with other instruments, and suggestions for the project's direction.

## Pull Request Guidelines

- **One change per PR.** Don't bundle unrelated edits.
- **Explain the "why."** Legal drafting is intentional. Your PR description should explain the problem the change solves and why your proposed language is the right fix.
- **Preserve formatting.** The SALE.docx has a specific formatting structure. If you're editing the .docx directly, try to maintain consistent styling.
- **Don't modify the form number or version.** Version increments are managed by the maintainer.

## What This Project Is Not

This is not a law firm and contributions do not create an attorney-client relationship. Contributors should not provide legal advice through Issues, PRs, or Discussions. All contributions are made under the project's CC0 1.0 Universal Public Domain Dedication.
